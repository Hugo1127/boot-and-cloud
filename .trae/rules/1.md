# 智能体开发规则

## 一、严格禁止项

&#x20;

1. **框架 / 依赖禁用**：禁用 Spring 系列及衍生依赖、第三方 IOC/AOP 框架，禁止集成成熟中间件，服务注册发现、配置中心等核心能力必须**从零手写实现**。
2. **开发规范禁用**：禁止代码混乱（**注释覆盖率≥70%**）、禁止无单元测试（**单元测试覆盖率≥80%**），禁止删减 JVM 调优、多线程优化等核心性能模块。

## 二、强制必须项

1. **开发实现**：所有核心逻辑**从零手写（禁止抄袭）**，遵循模块化、可插拔设计，完整实现 IOC/AOP、微服务核心能力，无功能缺失。
2. **过程管理**：无法一次性完成时，按规范记录剩余工作 / 问题到**TODO.md 文档，**禁止创建其他额外过程文档。
3. **环境 / 命令限定**：**所有执行命令仅适配 Windows 操作系统 + PowerShell 终端**；每个模块开发 / 修改完成后，必须执行编译命令验证通过。
4. **配置规范**：pom.xml 独立配置；公共代码统一抽离至公共模块；性能优化提供可复现测试用例。

## 三、TODO.md 编写规范

固定四部分结构，不可修改：

1. 已完成工作（简略概述）
2. 待完成模块（优先级、核心功能点、交付要求）
3. 待解决问题
4. 后续开发建议

## 四、补充要求

pom.xml独立配置；公共代码抽离至公共模块；性能优化提供可复现测试用例；面试考点绑定代码；可视化图表提供源文件+导出版。
